## Y read surfaces (the eden thesis read API)

Per the eden thesis, `Y` is whoever consumes eden's perception — human,
CLI user, LLM, or future autonomous agent. Y has four equivalent read
paths to the canonical `EdenPerception` report:

| Y form | path |
|---|---|
| HTTP / curl / browser | `GET /api/perception/:market` |
| Terminal | `eden perception <hk\|us>` |
| LLM (in-prompt context) | `perception` field in `initial_user_prompt` |
| LLM (mid-session tool) | `perception` tool in `tool_catalog` (`/api/perception/:market`) |

All four resolve to the same `EdenPerception` JSON, generated by
`PerceptionGraph::to_report()` and persisted as part of
`AgentSnapshot.perception`. Schema mirrored on the frontend in
`frontend/src/lib/api/types.ts::EdenPerception`.

Prefer these over the deprecated `/api/agent/:market/recommendations`
chain (RFC 8594 `Deprecation: true` header signals migration).
